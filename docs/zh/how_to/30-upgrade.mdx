# 补丁版本升级

由于需要修复漏洞或发布新功能，中间件将进行补丁版本升级。在补丁版本升级过程中，Pods 将逐个重启，这可能导致服务的短暂中断。因此，建议选择合适的时间窗口进行升级。当补丁版本可用时，建议尽快进行升级，以将最新修复应用于组件。

## 设置补丁版本升级策略

### 过程

<Tabs>
  <Tab label="Web 控制台">
    1. 转到实例详情页面。

    2. 点击 **补丁版本升级策略**。

    3. 选择 **手动** 或 **自动**。

    4. 点击 **更新** 完成设置。
  </Tab>
</Tabs>

## 逐个升级

### 先决条件

当补丁版本可用时，实例列表页面将显示升级图标，实例详情页面将显示升级提示。

### 过程

<Tabs>
  <Tab label="Web 控制台">
    1. 转到实例详情页面。

    2. 在升级提示旁点击 **立即升级**。

    3. 阅读升级提示以了解组件和插件的版本变更。

    4. 点击 **立即升级**。

    5. 等待实例状态更改为运行中，表示升级成功。
  </Tab>
</Tabs>

## 批量升级

### 先决条件

用户必须拥有平台管理员权限。

### 过程

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中点击 **升级管理**。

    2. 选择您需要操作的 **操作员**，并转到操作员详情页面。

    3. 选择您要升级的实例并点击 **批量升级**。

    4. 阅读升级提示以了解组件和插件的版本变更。

    5. 点击 **立即升级**。

    6. 等待实例状态更改为运行中，表示升级成功。
  </Tab>
</Tabs>
