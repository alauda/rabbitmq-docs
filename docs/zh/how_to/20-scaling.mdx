# 规格变更

随着数据量的增长，必须在资源使用接近存储配额之前及时进行扩展，以确保业务的持续可用性。

## 限制说明

1. 如果您已为当前实例设置了调度相关的配置，则副本数量的变化必须遵循当前的调度配置。

2. 建议将所有节点扩展到相同的规格，以增强集群的性能和可靠性。

3. 在更改 CPU 和内存资源时，请根据业务情况和资源可用性进行评估，以避免因资源不足或过多而导致实例扩展失败，进而可能导致实例无法正常启动。

## 步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏，点击 **RabbitMQ**。

    2. 点击 ***命名空间名称***。

    3. 点击 ***实例名称***。

    4. 在 **详细信息** 标签下，点击拓扑上方的 **更新规格**。

    5. 填入所需的副本数量和相应资源的新规格。

    6. 点击 **更新**。

       稍等片刻，您将看到拓扑中的 **规格** 已相应增加。
  </Tab>
</Tabs>
