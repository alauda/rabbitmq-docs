# 调度配置

灵雀云的 RabbitMQ 流服务支持在节点 taints 更改时动态更新 Pod 调度配置。您可以添加、修改或删除 Pod 容忍度，而不会中断正在进行的 RabbitMQ 服务。

## 注意事项

- **Pod 迁移限制**：在调度配置更新期间不支持 Pod 迁移。在修改 Pod 容忍度时，请确保更新后所选节点包括在更新之前属于部署的所有节点。

- **节点可用性要求**：如果您为调度配置了节点标签，匹配 Pod 容忍度和节点 taints 的节点数量必须大于或等于副本数。此外，更新后的节点选择必须与之前可用的节点相匹配，或者在原始节点集之外包含新节点，以确保调度配置能够成功更新。

## 步骤

<Tabs>
  <Tab label="网页控制台">
    1. 在左侧导航栏中，点击 **RabbitMQ**。

    2. 点击 ***命名空间名称***。

    3. 点击 ***实例名称***。

    4. 点击右上角的 **操作** 按钮 > **更新调度配置**。

    5. 根据需要修改调度配置：
       - 更新节点标签以优化合格节点的选择
       - 添加或修改容忍度以适应节点 taints 的变化
       - 调整亲和性规则以优化实例位置

    6. 点击 **更新** 以应用更改。
  </Tab>
</Tabs>

更新调度配置后，系统将在保持服务连续性的同时应用更改。操作员将在后续操作中创建或替换 Pods 时遵循新的调度参数。
