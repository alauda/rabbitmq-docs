# 用户管理

## 用户名和密码

<Directive type="info" title="提示">
**注意**：由于 RabbitMQ 默认用户 **guest** 的密码已公开，为确保环境安全，平台已禁用该用户。访问实例时可使用平台随机生成的用户。
</Directive>

在创建实例是，会创建一个随机的管理员账户，同时将该账户信息存储到名称满足 `<Instance name>-default-user` 的 secret 资源内。例如:

```yaml
apiVersion: v1
data:
  default_user.conf: ZGVmYXVsdF91c2VyPWFkbWluCmRlZmF1bHRfcGFzcz1hZG1pbjEyM18=
  password: YWRtaW4xMjNf
  provider: cmFiYml0bXE=
  type: cmFiYml0bXE=
  username: YWRtaW4=
kind: Secret
metadata:
  labels:
    app.kubernetes.io/component: rabbitmq
    app.kubernetes.io/name: rrrr
    app.kubernetes.io/part-of: rabbitmq
    prometheus.io/port: "15692"
    prometheus.io/scrape: "true"
  name: rrrr-default-user
type: Opaque
```

其中`username` 对应访问 RabbitMQ 的默认用户名，`password` 对应默认用户的密码。

<Directive type="warning" title="警告">
该 secret 是只读的，无法直接更新该 secret 来更新密码。否则会导致实例异常。
</Directive>

<Tabs>
<Tab label="CLI">

通过如下命令直接解析到默认用户的用户名和密码。

```bash
# 获取用户名
kubectl -n default get secret <instance name>-default-user -o jsonpath='{.data.username}' | base64 -d

# 获取密码
kubectl -n default get secret <instance name>-default-user -o jsonpath='{.data.password}' | base64 -d
```

</Tab>

<Tab label="Web Console">

在实例详细信息页，单击保密字典转至 Container Platform 查看用户名和密码。

</Tab>
