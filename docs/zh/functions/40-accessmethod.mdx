# 访问方式

使用生产者和消费者从集群内或集群外，或通过客户端将消息传递到 RabbitMQ。

## 如何获取访问地址

RabbitMQ 实例支持多种访问地址，以满足不同场景的需求。

### 集群内访问

集群内访问仅适用于 Kubernetes 网络内的内部访问。连接地址有两种选择：

- 内部路由（即通过 DNS）
- 内部路由 IP

这两种方法均允许访问 RabbitMQ 集群。

地址获取路径：实例详细信息 > 访问方式 > 集群内访问

### 集群外访问

集群外访问可用于对 Kubernetes 网络的外部访问，例如从另一个 Kubernetes 集群进行访问（也可以用于集群内部的访问，但需要根据防火墙规则确认）。

目前，NodePort 和 LoadBalancer 是支持的两种外部访问集群的方法。

地址获取路径：实例详细信息 > 访问方式 > 集群外访问
