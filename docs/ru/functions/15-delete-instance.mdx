# Удалить экземпляр

Удаление экземпляра не только удалит созданные ресурсы CR, но и удалит StatefulSet, Service, ConfigMap, Secret, PersistentVolumeClaim и другие ресурсы, созданные RabbitMQ на основе ресурсов CR.

## Процедура

<Tabs>
  <Tab label="CLI">
    Если экземпляр не был настроен для сохранения, или если вам нужно сохранить постоянные данные экземпляра, выполните следующую команду для удаления экземпляра:

    ```bash
    # удалить экземпляр rabbitmq напрямую
    $ kubectl -n default delete rabbitmqclusters <Имя экземпляра>
    ```

    По умолчанию удаление экземпляра не приведет к удалению созданного PVC. Вы можете использовать следующую команду для его удаления:

    ```bash
    $ kubectl -n default delete pvc -l app.kubernetes.io/component=rabbitmq,app.kubernetes.io/part-of=rabbitmq,app.kubernetes.io/name=<Имя экземпляра>
    ```
  </Tab>

  <Tab label="Веб-консоль">
    1. В левом навигационном меню нажмите **RabbitMQ**.

    2. Нажмите на ***имя пространства имён***.

    3. В списке экземпляров выберите экземпляр для удаления и перейдите на страницу сведений о экземпляре.

    4. В правом верхнем углу страницы **Сведения о экземпляре** нажмите кнопку **Действия** для раскрытия меню функций.

    5. Выберите **Удалить** в меню функций, и появится диалоговое окно подтверждения удаления.

    6. Для экземпляров с включенной сохранностью данных вы можете отметить "Удалить Persistent Volume Claim", чтобы освободить место, или оставить его незавершённым, чтобы сохранить данные.

    7. В открывшемся диалоговом окне введите имя экземпляра, который необходимо удалить.

    8. Нажмите **Удалить**.
  </Tab>
</Tabs>
