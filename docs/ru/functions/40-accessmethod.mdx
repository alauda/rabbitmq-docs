# Методы доступа

Используйте производителей и потребителей для отправки сообщений в RabbitMQ изнутри или снаружи кластера, или через клиент.

## Как получить адрес доступа

Экземпляры RabbitMQ поддерживают несколько адресов доступа, чтобы удовлетворить потребности различных сценариев.

### Доступ внутри кластера

Доступ внутри кластера доступен только для внутреннего использования в сети Kubernetes. Существует два варианта адресов подключения:

- Внутренний маршрут (т.е. через DNS)
- Внутренний IP-адрес маршрутизации

Оба метода обеспечивают доступ к кластеру RabbitMQ.

Путь получения адреса: Сведения об экземпляре > Методы доступа > Доступ внутри кластера

### Доступ вне кластера

Доступ вне кластера можно использовать для внешнего доступа к сети Kubernetes, например, для доступа из другого кластера Kubernetes (он также может использоваться для внутреннего доступа в рамках кластера Kubernetes, но это нужно подтвердить с учетом правил брандмауэра).

В настоящее время поддерживаются два метода для внешнего доступа к кластеру: NodePort и LoadBalancer.

Путь получения адреса: Сведения об экземпляре > Методы доступа > Доступ вне кластера
