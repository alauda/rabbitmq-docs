# Deployment

## Install Alauda Application Services Essentials Plugin

::: tip Optional Deployment
The plugin is a foundational component of Application Services for graphical interface services and does not need to be deployed if graphical interface services are not required.
:::

### Prerequisites

1. Download the Alauda Application Services Essentials installation package that matches your platform.
2. Use the platform's application listing capability to list Alauda Application Services Essentials to the ***global cluster***.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. In the left navigation bar, navigate to **App Store Management** > **Cluster Plugins**.
    2. Select the **global** cluster.
    3. Click the action button on the right side of the **Alauda Application Services Essentials** plugin > **Deploy**.
  </Tab>
</Tabs>

## Install Alauda Application Services RDS Framework Plugin

::: tip Optional Deployment
The plugin is a foundational component of Application Services for graphical interface services and does not need to be deployed if graphical interface services are not required.
:::

### Prerequisites

1. Download the Alauda Application Services RDS Framework installation package that matches your platform.
2. Use the platform's application listing capability to list Alauda Application Services RDS Framework to any cluster that wishes to use the graphical capabilities of Application Services.

### Steps

<Tabs>
  <Tab label="Web Console">
    1. Log in to the platform and go to the **Platform Management** page.
    2. In the left navigation bar, select **App Store Management** -> **Operator** to enter the **Operator Hub** page.
    3. Find **Alauda Application Services RDS Framework**, click **Deploy**, and enter the deployment page.

    Configuration Parameters:

    | **Parameter**   | **Recommended Configuration**                                 |
    | :-------------- | :----------------------------------------------------------- |
    | **Upgrade Strategy** | **Manual**: When a new version is available in Operator Hub, manual confirmation is needed to upgrade the Operator to the latest version. |
    | **Deployment Mode** | **Cluster**: All namespaces in the cluster share one Operator to create and manage instances, reducing resource usage.    |
    | **Channel**     | Default Channel is **stable**.                               |
    | **Namespace**   | Select **recommended namespace**: Automatically created if none exists.  |

    4. On the **Deploy Operator** page, select **Default Configuration**, click **Deploy**, and complete the deployment of **Alauda Application Services RDS Framework**.
  </Tab>
</Tabs>
