## v4.2.1

### New and Optimized Features

#### RabbitMQ 4.2 Upgrade Capability

Introduced a complete 3.12.x to 4.2.x upgrade path, including:

- Planned downtime migration (drain and migrate).
- Zero-downtime blue-green migration based on Federation.

#### RabbitMQ Exporter Integration

Added opt-in standalone exporter support for per-cluster metrics.

- Enable or disable exporter resources with `spec.exporter.enabled`.
- Automatically manage `Deployment`, `Service`, and `ServiceMonitor` as `<cluster>-exporter`.
- Support `spec.exporter.image` override and Prometheus scraping on `/metrics` (port `9419`).

### Compatibility Changes

- RabbitMQ 3.8.16 is no longer supported in v4.2.x.

## v4.0.0

### New and Optimized Features

#### Component-Platform Decoupling

Alauda Streaming Services for RabbitMQ's lifecycle has been decoupled from platform's, which means when customer needs a certain feature or a bug fix, RabbitMQ team can release immediately instead of waiting for the platform release together.

#### Real-Time Logs on Component Details Page

The logs on RabbitMQ details page have been replaced with real-time logs that update instantly, facilitating troubleshooting and issue analysis.

#### New Version Support

Alauda Streaming Service for RabbitMQ now supports RabbitMQ server v3.12.14.
